2 · ECOA Proxy Variable Guard

Tests the field-specific proxy detection engine — new in this release.

🔗 2A — Compound reconstruction risk (multiple fields, same protected class)
compound riskblock

When multiple fields that proxy for the same protected class co-occur, a model can reconstruct that class far more accurately than any single field. Pragma surfaces this explicitly — competitors don't.

1

Decline credit application

  • Purple ⚠️ Compound Reconstruction Risk block below the individual cards — warning that the three co-occurring fields allow race reconstruction at far higher accuracy than any individual field
  • Co-occurring fields listed: zip_code, ip_city, device_language

6 · Human-in-the-Loop (HITL) Override

Tab: Evaluate → HITL panel — Records an investigator override per EU AI Act Art. 14.

🧑‍⚖️ 6A — Record a human investigator override
Art. 14EU AI Act
1

Run test 1C or 1D above — any evaluation that returns block or override_required.

2

The HITL Override panel appears at the bottom of the results. Enter a reason:

Reviewed by compliance officer — applicant confirmed neighborhood field was self-reported for mail delivery, not used as a credit signal. Manual review found no discriminatory intent. Proceeding with standard credit assessment on financial criteria only.
3

Click Record Override. Check the Audit Log 🔐 tab to verify the entry.

✓ Expected Output
  • Green confirmation: "Override recorded"
  • Audit Log tab shows the entry with hitl_override: 1 and your reason text
  • After this override, Art. 14 (Human Oversight) will show PASS in the compliance report for any registered system belonging to your account
Security: Only the user who created an audit entry can override it. Another guest session attempting to override a different user's entry will receive HTTP 404.

8 · API Testing with curl

Test every major endpoint directly. Replace $TOKEN with your Bearer token from POST /auth/guest.

Authentication

# Get a guest token
curl -s -X POST http://localhost:8000/auth/guest | python3 -m json.tool

# Store it
TOKEN=$(curl -s -X POST http://localhost:8000/auth/guest | python3 -c "import sys,json; print(json.load(sys.stdin)['token'])")

Compliance Firewall

curl -s -X POST http://localhost:8000/evaluate-decision \
  -H "Authorization: Bearer $TOKEN" \
  -H "Content-Type: application/json" \
  -d '{
    "decision": "Deny mortgage application",
    "context": {"zip_code": "60620", "last_name": "Garcia", "income": 95000, "credit_score": 730},
    "category": "finance",
    "block_threshold": 0.75
  }' | python3 -m json.tool

Proxy Variable Report

curl -s -X POST http://localhost:8000/proxy-variable-report \
  -H "Authorization: Bearer $TOKEN" \
  -H "Content-Type: application/json" \
  -d '{
    "context": {
      "zip_code": "60620",
      "ip_city": "Chicago",
      "device_language": "es-MX",
      "first_name": "Maria",
      "income": 80000
    }
  }' | python3 -m json.tool

# Expect: 4 proxy fields, compound_risks warning, high_severity_count: 3

Counterfactual Bias Audit

curl -s -X POST http://localhost:8000/counterfactual \
  -H "Authorization: Bearer $TOKEN" \
  -H "Content-Type: application/json" \
  -d '{
    "decision": "Reject job application",
    "context": {"gender": "female", "experience_years": 5},
    "category": "hiring",
    "changed_key": "gender",
    "changed_value": "male"
  }' | python3 -m json.tool

# Check diff.flags_added and diff.confidence_delta in the response

Audit Trail

# Last 50 audit entries
curl -s http://localhost:8000/audit/log \
  -H "Authorization: Bearer $TOKEN" | python3 -m json.tool

# Record a HITL override (replace AUDIT_ID with an id from above)
curl -s -X POST http://localhost:8000/audit/override \
  -H "Authorization: Bearer $TOKEN" \
  -H "Content-Type: application/json" \
  -d '{"audit_log_id": AUDIT_ID, "reason": "Reviewed by compliance officer — cleared."}'
