Authentication Flow

Two paths: Google SSO or Guest. Both return a bearer token used for all subsequent requests.

Google SSO Flow

1

Load GIS SDK

Google Identity Services JS loaded on page init.

2

User Clicks Sign In

Google popup appears. User selects their account.

3

ID Token Returned

Google returns a signed JWT credential to the callback.

4

POST /auth/google

Token sent to backend. google-auth SDK verifies with Google's public keys.

5

Session Created

Backend returns opaque session token stored in localStorage.

Guest Flow

1

Click Continue as Guest

No credentials required. Immediate access.

2

POST /auth/guest

Backend generates a random anonymous ID and session token.

3

Token Stored

Same bearer token stored in localStorage. Expires in 24 hours.

4

Full Access

Guest users get full evaluate + PDF features. No name/picture shown.

Data & Privacy

No personal data stored. Only anonymous request metadata for usage analytics.

What We Store

  • Anonymous session ID (SHA-256 hash, not reversible)
  • Request timestamp
  • Context field names only (not values)
  • Decision word count (not the text)
  • LLM provider used (claude / openai / mock)
  • Confidence score & risk flag count

What We Never Store

  • User email or name
  • Google account details
  • The actual decision text
  • Context field values
  • IP addresses
  • Any personally identifiable information

SQLite Schema

CREATE TABLE request_logs ( id INTEGER PRIMARY KEY AUTOINCREMENT, anon_id TEXT, -- SHA-256 of session sub, not reversible timestamp TEXT, -- ISO 8601 provider TEXT, -- claude | openai | mock confidence_score REAL, risk_flag_count INTEGER, context_keys TEXT, -- JSON array of field names ["gender","role"] decision_words INTEGER -- word count, not the text );

LLM Orchestration

Automatic fallback chain ensures the service stays up even when an API is unavailable.

Primary — Claude

claude-opus-4-6 with Adaptive Thinking enabled. Streaming response. Falls back on rate limits, auth errors, or credit exhaustion.

Fallback — OpenAI

GPT-4o-mini. Activated when Claude is unavailable. Falls back on RateLimit or AuthenticationError.

Last Resort — Mock

Static structured response. Ensures the UI always renders. Labeled with provider: mock in response.

Deployment Guide

Three services, three platforms, all connected to the same backend.

1
Railway — Backend

Deploy FastAPI

Connect GitHub repo. Railway auto-detects Python via requirements.txt.

railway.app → New Project
→ Deploy from GitHub
→ ethical-ai-checker
→ Add env vars → Deploy

Required env vars: ANTHROPIC_API_KEY, OPENAI_API_KEY, GOOGLE_CLIENT_ID

2
Vercel — Web Frontend

Deploy Web App

Static HTML. Set root directory to frontend/ in Vercel dashboard.

vercel.com → New Project
→ Import ethical-ai-checker
→ Root: frontend/
→ Deploy

Update PROD_URL in frontend/index.html to your Railway URL.

3
Expo EAS — Mobile

Build Mobile App

Build and submit to App Store / Play Store via Expo EAS.

cd mobile
npm install -g eas-cli
eas login
eas build --platform ios
eas submit --platform ios

Update PROD_URL in mobile/src/config.ts to your Railway URL.
