Product & Architecture Overview

Pragma

A proprietary ethical reasoning model — purpose-built for AI governance. Learns from every decision. Improves with every user.

The Problem

Enterprises are deploying AI in hiring, lending, and operations — with no guardrails on ethical impact.

Invisible Bias

AI systems trained on historical data replicate systemic bias in hiring, loan approvals, and promotions — often without anyone noticing until litigation.

Regulatory Risk

EEOC, GDPR, and the EU AI Act require explainable, auditable decisions. Non-compliance carries fines up to 6% of global revenue.

No Tooling Exists

Ethics reviews today are manual, slow, and inconsistent. There is no automated system that applies structured moral reasoning at scale.

Model Distillation

Claude claude-opus-4-6 acts as the teacher, labelling ethical scenarios with structured JSON output. The Pragma model (Phi-3-mini LoRA adapter) learns to replicate this reasoning — at a fraction of the inference cost.

Feedback-Driven Training

Every thumbs up/down is stored by category and model version. collect_feedback.py identifies which categories have approval below 70% and auto-generates more targeted training examples for those areas.

Provider Phase-Out Plan

Claude and OpenAI are offline teachers only — used to bootstrap training data, not as the long-term production path. As the Pragma model matures, third-party LLM dependency drops to zero.

pip install transformers peft trl datasets

# Step 3 — Run the feedback flywheel
DATABASE_URL=postgresql://... \
ANTHROPIC_API_KEY=sk-ant-... \
python ml/collect_feedback.py --threshold 0.70
# Step 4 — Evaluate student vs teacher quality
HF_TOKEN=hf_... python ml/evaluate.py \
--student_repo yourname/pragma-ethics-v1
# Reports: parse rate, risk Jaccard, conf MAE

Use Cases

Real scenarios where the engine detects risk and provides actionable guidance.

Hiring Decision

Input decision:

"Reject job candidate" + context: gender=female, education=non-elite

Output:

bias flagged fairness flagged

Recommendation: Remove gender from evaluation criteria. Apply structured scoring on skills and experience only.

Loan Approval

Input decision:

"Reject loan application" + context: zip_code=90210, income=75000

Output:

discrimination flagged bias flagged

Recommendation: Zip code is a proxy for race and protected class. Replace with income, credit score, and debt-to-income ratio.

Promotion Decision

Input decision:

"Promote employee" + context: performance=excellent, years=3, gender=male

Output:

bias detected merit criteria present

Recommendation: Proceed, but remove gender from criteria. Decision appears merit-based — verify against team-wide promotion rates.

AI Policy Rollout

Input decision:

"Implement AI performance monitoring" + context: scope=keystrokes, consent=not required

Output:

transparency flagged harm potential

Recommendation: Require explicit employee consent. Disclose scope fully. Implement opt-out process and data retention limits.

Authentication Flow

Two paths: Google SSO or Guest. Both return a bearer token used for all subsequent requests.

Google SSO Flow

1

Load GIS SDK

Google Identity Services JS loaded on page init.

2

User Clicks Sign In

Google popup appears. User selects their account.

3

ID Token Returned

Google returns a signed JWT credential to the callback.

4

POST /auth/google

Token sent to backend. google-auth SDK verifies with Google's public keys.

5

Session Created

Backend returns opaque session token stored in localStorage.

Guest Flow

1

Click Continue as Guest

No credentials required. Immediate access.

2

POST /auth/guest

Backend generates a random anonymous ID and session token.

3

Token Stored

Same bearer token stored in localStorage. Expires in 24 hours.

4

Full Access

Guest users get full evaluate + PDF + feedback features. No name/picture shown.

Data & Privacy

No personal data stored. Feedback drives model improvement, not user profiling.

What We Store

  • Anonymous session ID (SHA-256 hash, not reversible)
  • Request timestamp
  • Context field names only (not values)
  • Decision word count (not the text)
  • Provider used (pragma / claude / openai / mock)
  • Confidence score, risk flag count, decision category
  • Feedback ratings (👍 / 👎) per category — for model retraining

What We Never Store

  • User email or name
  • Google account details
  • The actual decision text
  • Context field values
  • IP addresses
  • Any personally identifiable information

PostgreSQL Schema

-- Request metadata (no PII, no decision text) CREATE TABLE request_logs ( id SERIAL PRIMARY KEY, anon_id TEXT, -- SHA-256 of session sub, not reversible timestamp TEXT, -- ISO 8601 UTC context_keys TEXT, -- JSON array of field names ["gender","role"] decision_words INTEGER, -- word count only provider TEXT, -- pragma | claude | openai | mock confidence REAL, risk_count INTEGER, risk_categories TEXT, -- JSON array ["bias","fairness"] category TEXT -- hiring | finance | healthcare | ... ); -- Feedback for model retraining flywheel CREATE TABLE analysis_feedback ( id SERIAL PRIMARY KEY, anon_id TEXT, timestamp TEXT, rating INTEGER, -- 1 = thumbs up, -1 = thumbs down category TEXT, provider TEXT, model_version TEXT, -- e.g. "pragma-v1", "unknown" confidence REAL, risk_categories TEXT -- JSON array );
